ASLClient

ASLClient instances maintain a client connection to the ASL daemon, and can used to perform logging and to perform log search queries.

Note: Because the underlying client connection is not intended to be shared across threads, each ASLClient has an associated GCD serial queue used to ensure that the underlying ASL client connection is only ever used from a single thread.

    public init(sender: String? = nil, facility: String? = nil, filterMask: Int32 = ASLPriorityLevel.Debug.filterMaskUpTo, useRawStdErr: Bool = true, options: Options = .NoRemote)

    Parameters

    sender

    Will be used as the ASLMessageKey value for the .Sender key for all log messages sent to ASL. If nil, ASL will use the process name.
    facility

    Will be used as the ASLMessageKey value for the .Facility key for all log messages sent to ASL. If nil, ASL will select a default.
    filterMask

    Specifies the priority filter that should be applied to messages sent to the log.
    useRawStdErr

    If true, messages sent through the ASLClient will be mirrored to standard error without modification. Note that this differs from the behavior of the .StdErr value for the ASLClient.Options parameter, which performs some escaping and may add additional text to the message.
    options

    An ASLClient.Options value specifying the client options to be used by this new client. Note that if the .StdErr value is passed and rawStdErr is also true, the behavior of rawStdErr will be used, overriding the .StdErr behavior.

  • search(_:callback:)

    Asynchronously reads the ASL log, issuing one call to the callback function for each relevant entry in the log.
